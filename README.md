# Подборка фильмов — Лабораторная (Вариант 4)

Проект содержит фронтенд и два простых сервисa для выполнения основных требований ЛР3 (вариант 4 — рекомендации фильмов).

Что реализовано (минимальные требования):
- `service-b` — API, возвращает топ‑N фильмов по жанру: `GET /recommendations?genre=&limit=`.
- `service-a` — клиент/нагрузчик, отправляет запросы к `service-b` и собирает простую статистику латентностей.
- фронтенд для локального просмотра данных и фильтрации: `index.html`, `app.js`, `movies.json`.

Запуск и проверка локально:

```powershell
cd "C:/Users/Пользователь/Desktop/project js/lab-3-js"
npm install
# Запустить сервис B (API)
npm run start:b
# В другом терминале запустить генератор (пример):
node service-a/generator.js --genre "Sci-Fi" --requests 200 --concurrency 10 --host http://localhost:3000 --limit 5
```

Docker (рекомендуемый — запускает фронтенд + API):

```powershell
# собрать и запустить nginx-фронтенд + service-b
docker-compose up --build -d

# открыть фронтенд http://localhost:8080
# API доступен на http://localhost:3000
```

Файлы:
- `service-b/index.js` — реализация API (использует `movies.json`).
- `service-a/generator.js` — генератор нагрузки и простой бенчмаркинг.
- `movies.json` — набор фильмов для примера.

Замечание по методичке:
реализация `service-b` намеренно содержит неэффективности (дублирование массивов, двойная сортировка) — это соответствует требованиям ЛР3 (сервис должен быть профилируемым/оптимизируемым). Если нужно, могу сделать оптимизированную версию и измерить улучшение.

Если хотите Java‑версию `service-b` для работы с JFR/JMC — сделаю по запросу.
